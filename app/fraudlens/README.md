# fraudlens.

Upload a transactions CSV and get an interactive fraud-intelligence dashboard:
a world map of where the risk sits, live-tunable detection rules with plain-
English explanations for every flag, and a threat-ranked list of accounts.
Everything runs client-side — the file never leaves the browser.

## Run it

```
npm install
npm run dev
```

Then open the printed URL and either drop in your own CSV or click
"Try the demo dataset". A committable sample lives at
`public/sample_transactions.csv` (synthetic, generated by the same seeded
generator as the demo button — no real data).

`npm run build` produces a static bundle in `dist/` that can be served from
any static host. For GitHub Pages set `GITHUB_PAGES=<repo-name>` at build time
so asset paths resolve under `/<repo-name>/`.

## What it expects

Any CSV with columns for user id, transaction type, amount (minor units),
currency and user country. Merchant country, KYC status, birth year and a
fraud label are optional — the dashboard degrades gracefully when they are
missing. Column names are auto-guessed and can be remapped by hand before
the analysis runs.

## How it works

- A Web Worker streams the CSV with PapaParse and reduces it to per-user and
  per-country aggregates on the fly, so a 700k-row file parses in seconds
  without holding raw rows in memory.
- Six explainable rules run over the aggregates (mule pass-through, card
  testing, merchant-country spread, amount outliers, KYC anomalies and
  corridor concentration). Each fired rule produces a reason with the actual
  numbers behind it, and every threshold is a slider — changes re-flag the
  dataset live.
- Flagged accounts get a 0–100 threat score that mixes rule severity,
  volume, repeat offences and corridor dominance; when the dataset carries a
  fraud label, the score also weighs the confirmed-fraud rate and a quality
  card reports how the rules line up with the labels (precision, recall and
  the full confusion matrix).

Amounts are converted to GBP with a fixed snapshot of FX rates; currencies
outside the table (including crypto) are excluded from monetary figures and
reported as a footnote. Ages are anchored to 2020 because the data carries
no timestamps.
